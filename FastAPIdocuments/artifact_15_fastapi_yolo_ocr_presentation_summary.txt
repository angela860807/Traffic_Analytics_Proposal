FastAPI 기반 YOLO/OCR 교통 분석 기능 중간 점검 요약

이번 개발에서는 라즈베리파이 또는 로컬 영상에서 전달되는 차량 프레임을 FastAPI 서버가 수신하고, AI 모델을 이용해 차량과 번호판을 탐지한 뒤 OCR 결과를 Spring Boot 백엔드와 PostgreSQL에 저장하는 흐름을 구현했다. 전체 구조는 Vue 프론트엔드, FastAPI AI 서버, Spring Boot API 서버, PostgreSQL DB로 나뉘며, FastAPI는 영상 프레임 분석과 이미지 산출물 저장을 담당한다.

FastAPI에는 `/api/detections/stream-frame` 엔드포인트를 구성했다. 이 API는 영상 프레임을 multipart 형식으로 받아 차량 YOLO 모델을 실행하고, bbox가 감지되면 차량 이벤트를 시작한다. 차량이 일정 프레임 동안 사라지거나 최대 이벤트 시간이 지나면 이벤트를 종료하고, 그동안 모아둔 후보 프레임 중 가장 좋은 best frame을 선택한다. best frame은 confidence, blur score, 중앙 위치, bbox 크기 등을 기준으로 점수화한다.

차량 이벤트가 종료되면 번호판 YOLO 모델로 번호판 영역을 찾고, 번호판 crop 이미지를 만든 뒤 PaddleOCR을 통해 번호판 문자열을 인식한다. OCR 전처리는 resize, CLAHE, sharpening, Otsu, adaptive threshold 등 여러 variant를 만들어 가장 좋은 결과를 고르는 방식으로 구성했다. 최근에는 숫자만 긴 후보가 잘못 선택되는 문제를 개선하기 위해 한국 번호판 형식에 가까운 결과를 우선하도록 후처리 점수를 추가했다.

이미지 산출물은 `fastapi-server/storage/detections` 아래에 날짜별로 저장된다. 기본 frame 이미지, 번호판 crop 이미지, OCR 전처리 이미지가 저장되며, 시연용으로 best frame을 다시 찾아가 원본 해상도 기준 차량 crop을 만든 뒤 high-res OCR을 재시도하는 흐름도 구현했다. 이 crop은 최근 수정으로 정사각형 형태가 되도록 보정했다.

Spring Boot 연동은 FastAPI가 분석 결과를 `/api/v1/detection-logs`로 전송하는 방식이다. 전송 시 `X-Internal-Api-Key` 헤더를 사용하며, 정상 인식은 `FLOW_EVENT_CREATED`, 중복 번호판은 `DUPLICATE_SKIPPED`, 번호판 미인식은 `OCR_FAILED` 상태로 저장한다. 과속 탐지 결과가 있는 경우에는 flow event 생성 후 speed violation API로 별도 저장한다.

시연 과정에서는 OpenCV preview 창에서 FastAPI 응답 bbox를 영상 위에 표시하도록 했다. 학원 PC에서는 추론 속도가 느려 async 업로드 사용 시 중간 프레임이 드롭되어 차량 bbox를 놓치는 문제가 있었다. 이를 해결하기 위해 시연용 명령어에서는 `--no-async-upload`, 낮은 fps, 적절한 upload scale을 사용하여 영상은 조금 끊기더라도 bbox 검출률을 우선하도록 조정했다. 그 결과 차량 5대 중 대부분을 안정적으로 잡는 수준까지 개선했다.

현재 한계도 명확하다. `/stream-frame` 요청 안에서 차량 탐지, 이벤트 종료, 번호판 탐지, OCR, DB 저장까지 모두 처리하기 때문에 느린 PC에서는 실시간성이 떨어진다. 또한 차량 YOLO 모델은 도로 그림자나 차선 영역을 차량으로 오인하는 경우가 있어 ROI, bbox 면적, 가로세로비 기반 후처리가 더 필요하다. 실무형으로 확장하려면 실시간 bbox 응답과 OCR/DB 저장을 분리하고, ByteTrack 같은 tracking을 도입해 차량 단위 이벤트를 안정적으로 관리하는 구조로 개선하는 것이 다음 목표다.

정리하면 이번 단계에서는 YOLO 차량 탐지, 번호판 탐지, PaddleOCR 인식, 이미지 저장, Spring DB 연동, preview 시연 명령어까지 end-to-end 흐름을 구현했다. 아직 실시간 처리 구조와 모델 후처리에는 개선 여지가 있지만, AI 기반 교통 분석 시스템의 핵심 파이프라인은 실제 시연 가능한 형태로 연결된 상태다.
