# TAS-PM FastAPI 예지보전 작업일지

작성일: 2026-06-14

## 1. GitHub 한 줄 제목

```text
feat(fastapi): 예지보전 AI 계약·artifact 검증·관측성 및 자동 테스트 보강
```

## 2. 작업 목표

AI 모델 파트와 Spring Boot 파트의 최종 산출물이 없어도 FastAPI에서 준비 가능한
범위를 구현했다.

- 예지보전 기존 기능의 자동 테스트 확보
- FastAPI DTO와 AI 패키지 계약 경계 분리
- LSTM artifact 검증과 시작 시 1회 초기화
- 내부 API 오류, request ID, body 제한 표준화
- endpoint, detector, Spring 전송 metric 제공
- Docker의 AI wheel·artifact 주입 경로 준비

## 3. 자동 테스트

추가 파일:

```text
fastapi-server/tests/predictive/
  test_camera_health_pipeline.py
  test_predictive_adapter.py
  test_predictive_api.py
```

검증 범위:

- 상태 window의 `COMPLETE`, `PARTIAL`, `INSUFFICIENT`
- OCR 시도 0건의 `ocrFailRate=null`
- 동일 window의 `idempotencyKey`
- Spring 재시도 시 payload와 `X-Request-Id`
- queue 포화 시 drop counter
- FastAPI DTO에서 AI 입력 DTO로 변환
- `LEARNING` 상태의 SHADOW 추론 생략
- LSTM 결과의 `shadowCandidates` 분리
- artifact SHA-256과 feature 순서
- 공통 validation 오류와 body 크기 제한
- metric endpoint 내부 인증
- 두 탐지 endpoint fake AI 인계 스모크

작성 이유:

외부 패키지가 없더라도 FastAPI 소유 범위의 회귀를 검출하고, 향후 실제 AI package
연결 시 fake module만 실제 package로 교체할 수 있도록 만들었다.

## 4. AI Adapter 경계

변경 파일:

```text
app/services/predictive_detector_adapter.py
```

FastAPI 요청 DTO를 다음 AI 소유 입력 타입으로 변환한다.

```text
RuleDetectionInput
DegradationDetectionInput
ModelPredictionInput
```

작성 이유:

FastAPI Pydantic 객체를 그대로 AI 함수에 전달하면 AI 패키지가 FastAPI 구현에
결합된다. 명시적 변환을 적용해 양쪽 계약이 다르면 호출 초기에 실패하도록 했다.

추가 정책:

- `baselineStatus=LEARNING`이면 모델 추론을 생략한다.
- SHADOW 결과를 운영 후보에 병합하지 않는다.
- AI 결과의 Enum, `cameraId`, `evaluatedAt`을 재검증한다.
- package version과 detector version을 분리한다.

## 5. Artifact 검증

추가 파일:

```text
app/services/predictive_artifacts.py
```

검증 내용:

- 필수 artifact 파일 존재
- SHA-256 일치
- feature schema version
- 8개 feature 이름과 순서
- lifespan 시작 시 한 번만 초기화

작성 이유:

요청마다 artifact를 다시 읽지 않고, 잘못된 모델 또는 전처리 schema로 추론하는
문제를 서버 시작 단계에서 발견하기 위해서다.

## 6. 오류·보안·Request ID

추가 또는 변경:

```text
app/core/middleware.py
app/core/request_context.py
app/core/exceptions.py
```

적용 내용:

- `X-Request-Id` 생성 및 전파
- validation 오류를 `400 INVALID_REQUEST`로 변환
- `fieldErrors` 생성
- 내부 API body 크기 제한
- detector 실행 로그에 request ID 연결

작성 이유:

Spring과 FastAPI의 요청 로그를 연결하고, FastAPI 기본 `422` 형식을 프로젝트 오류
계약과 맞추기 위해서다.

## 7. 관측성

추가 endpoint:

```http
GET /internal/v1/anomaly-detection/metrics
```

노출 값:

```text
endpoint 호출·오류·처리 시간
detector 호출·오류·처리 시간
queue 크기
전송 enqueue·성공·retry·최종 실패·drop
최근 Spring RTT
```

작성 이유:

기존 counter는 프로세스 내부에만 있어 queue 포화와 Spring 전송 장애를 확인하기
어려웠다. 내부 API key가 필요한 endpoint로 운영 상태를 조회하도록 했다.

## 11. 현재 한계

- 전송 queue는 메모리 기반이다.
- FastAPI 재시작 시 미전송 샘플은 복구되지 않는다.
- 최대 재시도 실패 샘플은 자동 재적재되지 않는다.
- runtime metric은 재시작 시 초기화된다.
- AI 패키지 설치 전 detector health는 `DEGRADED`다.
