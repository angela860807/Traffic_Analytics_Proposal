# TAS-PM FastAPI 검수 보완 작업일지

작성일: 2026-06-15

## 1. 작업 범위

- `MISSING`, `INVALID` artifact 상태에서 LSTM AutoEncoder SHADOW 추론 차단
- Docker health check가 HTTP 200뿐 아니라 detector health 본문의 `status`까지 확인
- 위 동작을 고정하는 adapter 회귀 테스트 추가

## 2. Artifact 추론 차단

`PredictiveDetectorAdapter`는 SHADOW 추론 전에 artifact 상태를 한 번 초기화하고
검증한다. SHADOW 추론은 검증이 완료된 `READY` 상태에서만 실행한다.

- `READY`: SHADOW 추론 허용
- `NOT_CONFIGURED`: SHADOW 추론 생략
- `MISSING`: SHADOW 추론 생략
- `INVALID`: SHADOW 추론 생략

artifact가 없거나 검증에 실패해도 Rule·추세 detector 결과는 반환한다. SHADOW
결과만 빈 배열로 유지하며, request ID·camera ID·artifact 상태와 오류를 경고
로그로 남긴다.

## 3. Docker Health Check

`/internal/v1/anomaly-detection/health`는 상세 상태 전달을 위해 기존처럼 HTTP 200을
반환한다. Docker health check는 응답 JSON을 읽고 `status == "UP"`인 경우에만
성공하도록 변경했다.

따라서 AI 패키지 미설치, detector 비활성, artifact 누락·오류로 응답이
`DEGRADED` 또는 `DOWN`이면 컨테이너도 unhealthy로 판정된다.

## 4. 테스트

추가한 회귀 조건:

- artifact가 설정되지 않으면 `NOT_CONFIGURED`이며 SHADOW 함수를 호출하지 않는다.
- 필수 artifact 디렉토리가 없으면 `MISSING`이며 SHADOW 함수를 호출하지 않는다.
- artifact 디렉토리가 불완전하면 `INVALID`이며 SHADOW 함수를 호출하지 않는다.
- 유효한 6종 artifact fixture가 `READY`일 때만 SHADOW 결과를 반환한다.
- 세 오류 상태 모두 Rule·추세 결과는 유지하고 health는 `DEGRADED`다.

Docker 이미지는 이 작업에서 빌드하지 않는다. Python 테스트, compileall,
`docker compose config --quiet`만 실행한다.

## 5. FastAPI 단독 작업 마무리

- 서버 시작 시 AI 패키지 설치 여부·버전, artifact 상태, feature schema version,
  검증 파일 목록을 한 번만 로그로 출력한다.
- 프레임·OCR 상태 수집 중 예외가 발생해도 영상 분석은 계속하고 오류 로그만
  남기도록 부가 기능 장애를 격리한다.
- FastAPI 컨테이너에 CPU `2.0`, 메모리 `4g` 제한을 설정한다.
- 시작 로그 1회 출력과 상태 수집 장애 격리 단위 테스트를 추가한다.
