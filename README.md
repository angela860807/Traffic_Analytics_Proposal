# 🚦 교통 관리 시스템 (Traffic Management System)

본 프로젝트는 AI 분석 기반의 교통 탐지 로그 관리 및 흐름 분석 시스템입니다.

## 🛠 실행 전 필수 확인 사항

이 프로젝트는 로컬 환경의 PostgreSQL 데이터베이스와 연동되어 있습니다. 애플리케이션 실행 및 테스트 전 아래 설정을 반드시 확인하세요.

### 1. 데이터베이스 서버 실행 (application.yml 기준)
테스트와 빌드를 수행하기 위해 PostgreSQL 서버가 구동 중이어야 합니다.
- **DB 종류**: PostgreSQL
- **주소**: `localhost:5432`
- **데이터베이스명**: `traffic`
- **계정 정보**: ID(`postgres`) / PW(`1004`)

### 2. 빌드 및 테스트 시 주의사항 (피드백 2-4 반영)
로컬 DB가 실행되지 않은 상태에서 빌드를 시도하면 테스트 단계에서 오류가 발생합니다.
- **정상 빌드/테스트**: `./gradlew build` (또는 `./gradlew test`)
- **DB 없이 빌드만 수행**: `./gradlew build -x test` (테스트 제외 빌드)

---

## 🔍 프로젝트 핵심 로직 준수 사항
이 프로젝트는 코드 품질 향상을 위해 다음 가이드를 준수합니다.
- **예외 처리**: AI 응답 값 오류 시 `502 Bad Gateway` 성격의 `BusinessException`을 던집니다.
- **데이터 방어**: Enum 변환 시 `of()` 메서드를 통해 잘못된 값이나 null 입력에 대비합니다.
- **성능 최적화**: AI 서버 호출 시 DB 트랜잭션이 길게 유지되지 않도록 트랜잭션 범위를 최소화합니다.
